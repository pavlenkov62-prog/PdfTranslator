# Правила разработки PDFTranslator

## Цель проекта

Создать интерактивный редактор перевода PDF, сохраняющий структуру
оригинального документа и позволяющий вручную корректировать результат.

------------------------------------------------------------------------

## Основные принципы

-   Проект всегда должен оставаться в рабочем состоянии.
-   Разработка ведется небольшими, законченными этапами.
-   После завершения каждого логического этапа создается коммит.
-   Не объединять несколько крупных изменений в одном этапе.

------------------------------------------------------------------------

## Архитектура

Последовательность обработки документа:

1.  Загрузка PDF.
2.  Извлечение структуры страницы.
3.  Извлечение текстовых блоков и графики.
4.  Перевод текста.
5.  Отрисовка результата.
6.  Редактирование пользователем.
7.  Экспорт.

------------------------------------------------------------------------

## Типы текстовых блоков

### SingleLine

Используется для:

-   заголовков;
-   подписей;
-   ссылок;
-   коротких технических надписей.

Поведение:

-   никогда не переносится;
-   после перевода автоматически расширяется вправо;
-   ширина рассчитывается по оригинальному тексту и его bbox;
-   Undo восстанавливает исходный размер.

### Paragraph

Используется для обычного текста.

Поведение:

-   допускает переносы;
-   в дальнейшем поддерживает ручное изменение размеров;
-   после изменения размеров текст автоматически перекомпоновывается.

------------------------------------------------------------------------

## Координаты

Каждый TextBlock хранит:

-   bbox --- исходные координаты PDF;
-   current_bbox --- текущие координаты после автоматических или ручных
    изменений.

При отображении используются current_bbox.

------------------------------------------------------------------------

## Правила перевода

-   Пустые блоки не переводятся.
-   Блоки, содержащие только цифры, не переводятся.
-   Односимвольные блоки пропускаются.
-   Блоки без букв не переводятся.

------------------------------------------------------------------------

## Интерфейс

-   Информация о документе отображается сверху.
-   Информация о странице обновляется автоматически.
-   Инспектор блока обновляется автоматически при выборе блока.
-   Лишние кнопки удаляются после автоматизации их функций.

------------------------------------------------------------------------

## Разработка

При изменении кода:

-   изменять минимально необходимый объем;
-   сначала сделать рабочее решение;
-   затем улучшать архитектуру;
-   не использовать магические коэффициенты, если их можно вычислить
    автоматически.

------------------------------------------------------------------------

## Коммиты

Каждый коммит должен содержать:

-   одно законченное изменение;
-   понятный заголовок;
-   краткое описание выполненной работы.

------------------------------------------------------------------------

## Долгосрочный план

1.  Завершить SingleLine.
2.  Реализовать ручное изменение размеров Paragraph.
3.  Автоматическая перекомпоновка Paragraph.
4.  Экспорт измененных bbox.
5.  Редактирование блоков мышью.
6.  Обработка графики.
7.  Финальный экспорт PDF.
